Why it exists

Model access is messy. Your apps should not care.

Most AI tools want one base URL and one API key. Real life is different: Google has a Gemini format, Groq and Mistral are OpenAI-compatible, Cloudflare needs account-scoped keys, local servers move around, and free tiers get rate-limited.

LLMHarbor gives you one stable local endpoint while it handles provider credentials, model registration, fallback order, health checks, and request analytics behind the scenes.

Run, configure, and operate LLMHarbor

Quickstart

  1. Install with the curl command above.
  2. Start the app with llmharbor start or run from the repo with npm run dev.
  3. Open the local dashboard and create a client key on the Keys page.
  4. Add at least one provider key, then run Check all.
  5. Point any OpenAI-compatible client at http://127.0.0.1:3001/v1.
curl http://127.0.0.1:3001/v1/chat/completions \
  -H "Authorization: Bearer $LLMHARBOR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"auto","messages":[{"role":"user","content":"hello"}]}'

Core concepts

Client API keys
Keys used by your apps to call LLMHarbor. They are separate from provider credentials and can be revoked independently.
Provider keys
Encrypted upstream credentials for Google, Groq, Mistral, OpenRouter, custom endpoints, and other providers.
Models
Provider model rows with limits, context windows, display names, and routing metadata.
Fallback chain
The ordered list used when model: auto or a requested model needs a healthy backup.

Providers and custom endpoints

Built-in providers appear first in the Keys and Models pages. Custom OpenAI-compatible endpoints can be added from Keys and then populated with models from Models. Provider ids used by bulk import match this same visible order.

Routing and fallback

LLMHarbor routes requests through enabled models with healthy credentials. The fallback page lets you reorder priorities, pause rows, and shape the model chain. The router tracks provider errors and avoids wasting calls on disabled or unhealthy routes.

Security model

The dashboard/control-plane API is loopback-only by default. Provider keys are encrypted before storage and never returned in plaintext from listing endpoints. Do not expose the server directly to the public internet; if you bind to 0.0.0.0, put it behind your own VPN, firewall, reverse proxy, and authentication layer.

Operations

  • Use HOST=127.0.0.1 for local-only operation.
  • Use PORT=3001 or another valid port between 1 and 65535.
  • Run npm test, npm run build, and npm audit before deploying changes.
  • Back up the SQLite database if you rely on stored provider credentials.
